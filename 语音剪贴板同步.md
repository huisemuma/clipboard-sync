# 语音剪贴板同步

> 设计文档

## 出发点
豆包输入法的语音识别非常好，但电脑端没有（或很垃圾）。目前的工作流是：手机语音输入→发到笔记平台→同步→电脑复制粘贴，步骤太繁琐。

## 核心需求
手机语音输入完毕发送后，自动同步到电脑剪贴板，只需粘贴即可。

## 技术方案

### 架构：网页套壳 + 本地监听
- **手机端**：网页应用（PWA 或普通网页），最通用，iOS/Android 都能用，不需要开发原生 App
- **电脑端（Mac）**：轻量后台服务，监听消息并写入系统剪贴板
- **中间通道**：WebSocket 长连接，实时推送

### 方案对比

| 方案 | 手机端 | 电脑端 | 连接方式 | 优缺点 |
|------|--------|--------|----------|--------|
| A. 局域网直连 | 网页 | Node/Python 服务 | 同一 WiFi 下 WebSocket | 最快、无延迟、无服务器成本；但离开局域网不可用 |
| B. 中继服务器 | 网页 | 客户端 | 通过云端 WebSocket 中继 | 跨网可用；需要一台服务器 |
| C. 推送服务 | 网页 | 客户端 | Firebase/APNs 推送 | 跨网、省电；但有延迟，配置复杂 |

**推荐：先做方案 A（局域网），够用后再扩展 B。** 90% 的使用场景是手机和电脑在同一 WiFi 下。

### 技术要点
1. **Mac 写入剪贴板**：`pbcopy` 命令或 Node.js `clipboardy` 库，无权限限制
2. **手机网页输入**：标准 textarea + 发送按钮，豆包输入法在任何输入框都能用
3. **设备配对**：首次扫码（电脑显示二维码，手机扫描获取 IP + 端口），之后自动连接
4. **断线重连**：WebSocket 断了自动重连，体验流畅

### 技术难点
1. **局域网服务发现**：手机怎么找到电脑？→ 扫码解决，或 mDNS/Bonjour 自动发现
2. **HTTPS 要求**：部分浏览器要求安全上下文才能用某些 API → 局域网可用自签名证书或 HTTP（Chrome 允许局域网 HTTP）
3. **iOS Safari 限制**：后台标签页 WebSocket 会断 → 用户需保持页面在前台，或改用 Service Worker（有限支持）
4. **跨网场景**：需要中继服务器，增加部署复杂度

### MVP 最快路径（1-2天）
1. Mac 跑一个 Node.js WebSocket 服务（10行代码）
2. 收到消息 → `pbcopy` 写入剪贴板
3. 手机浏览器打开 `http://电脑IP:端口`，textarea 输入，点发送
4. Done. 语音输入 → 点发送 → 电脑 Cmd+V

## 跨网场景（公司环境）

典型使用场景：手机走 4G/5G，电脑走公司 WiFi，两边不同网络，**必须中转**。

### 无账号配对方案

| 方案 | 体验 | 安全性 | 备注 |
|------|------|--------|------|
| **6位配对码** | 电脑生成码，手机输入 | 一次性，用完即失效 | 类似投屏配对，体验最干净 |
| **二维码** | 电脑显示，手机扫描 | 同上 | 公司电脑屏幕显示二维码可能奇怪 |
| **自定义暗号** | 用户自设暗号，双端输入同一暗号配对 | 碰撞风险，加随机后缀降低 | 最简单，但有碰撞可能 |

**推荐：6位配对码**，体验最干净，安全性够用。

### 中继服务选择

- **Cloudflare Workers**：免费额度足够，支持 WebSocket，无需自己运维，延迟可控
- 备选：Ably / Pusher 免费 tier

最终链路：`手机网页 → WebSocket → CF Worker → WebSocket → 电脑客户端 → pbcopy`

### 隐私方案（面向分享给朋友场景）

每条消息都走服务器中转，隐私担忧是真实的。

**推荐：WebRTC直连优先 + 加密中转降级**

1. **WebRTC P2P打洞（优先）**：配对成功后先尝试 WebRTC 直连，跨网也有概率打通。打通则数据完全不经过第三方
2. **E2E加密**：配对时交换密钥，手机端加密、电脑端解密，服务器只转发密文，看不到内容（Web Crypto API 原生支持）
3. **消息不落盘**：服务器纯转发，不写数据库，断线后无痕迹

打不通 WebRTC 就降级到加密中转，对用户体验透明。

自己用的话无所谓；**公开发布必须有E2E加密+不落盘，否则别人不敢用。**

## 扩展可能
- 支持双向同步（电脑剪贴板也能推到手机）
- 历史记录列表
- 多设备支持
- 如果做成产品分享给朋友，可以考虑 Electron 打包 Mac 端 + 网页 PWA 手机端

## 备注
- iOS/Android 用户都有，所以网页方案是最通用的选择
- 不需要自己做语音识别，手机输入法已经处理好了，只传文字
