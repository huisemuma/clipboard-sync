# 剪贴板同步 (Clipboard Sync)

手机语音输入 → 点发送 → 电脑 Cmd+V 直接粘贴。

## 为什么做这个？

手机上的语音输入法（豆包、讯飞等）识别率已经非常高了，说一段话基本能准确转成文字。但电脑端的语音输入要么没有、要么很垃圾——macOS 自带的听写功能识别率低、响应慢，而且不支持第三方输入法的语音能力。

于是就出现了一个很蠢的工作流：手机语音输入 → 发到微信/备忘录 → 电脑上打开同一个 App → 复制 → 粘贴。明明手机已经把语音转好了文字，却要绕一大圈才能到电脑剪贴板。

这个工具就是把中间的弯路全砍掉：**手机说完话，点一下发送，电脑直接 Cmd+V**。整个过程不到 1 秒，不需要装 App，不需要登录账号，手机浏览器打开就能用。

## 工作原理

```
手机浏览器 ──WebSocket──→ Mac Node.js 服务 ──pbcopy──→ 系统剪贴板
```

- Mac 端运行一个轻量 Node.js 服务（端口 9898）
- 手机在同一 WiFi 下用浏览器打开服务地址
- 手机输入文字（语音/打字均可）→ 点发送 → 通过 WebSocket 实时推送到 Mac
- Mac 收到后自动写入系统剪贴板，Cmd+V 即可粘贴
- 配合豆包输入法等语音输入法效果最佳——手机语音识别能力远超电脑端

## 快速开始

### 前置要求

- macOS（Apple Silicon / Intel 均可）
- Node.js（v18+）
- 手机和电脑在同一 WiFi

### 启动

**方式一：双击启动（推荐）**

双击 `启动剪贴板同步.command` 文件，首次运行会自动安装依赖。

**方式二：命令行启动**

```bash
cd clipboard-sync
npm install   # 首次需要
node clipboard-sync.mjs
```

启动后终端会显示：
```
📋 剪贴板同步已启动
   手机浏览器打开: http://192.168.x.x:9898
   然后语音输入 → 点发送 → Mac 直接 Cmd+V
```

### 使用

1. 手机浏览器打开终端显示的地址
2. 页面顶部绿点亮起 = 已连接
3. 在输入框里语音输入或打字
4. 点「发送到电脑」（或按 Enter）
5. 电脑上 Cmd+V 粘贴

用完关终端窗口即可。

## 技术细节

| 项目 | 说明 |
|------|------|
| 运行时 | Node.js + ws 库 |
| 端口 | 9898 |
| 协议 | HTTP（页面）+ WebSocket（实时通信） |
| 剪贴板写入 | macOS `pbcopy` 命令 |
| 前端 | 内嵌 HTML，无需额外文件，暗色主题 |
| 断线重连 | WebSocket 断开后 1.5s 自动重连 |
| 依赖 | 仅 `ws`，无其他第三方依赖 |

## 局限性

- **仅限局域网**：手机和电脑必须在同一 WiFi 下
- **单向同步**：目前只支持 手机→电脑，不支持反向
- **iOS Safari**：需保持页面在前台，后台会断开 WebSocket
- **无认证**：同一局域网内任何人知道地址都能发送（适合个人/家庭使用）

## 跨网方案（规划中）

公司场景下手机走 4G/5G、电脑走公司 WiFi，需要中转：

- **Cloudflare Workers** 做 WebSocket 中继
- **6位配对码** 无账号配对（类似投屏配对）
- **WebRTC 直连优先**，打不通再降级到加密中转
- **E2E 加密 + 不落盘**（面向分享给朋友的场景）

详见 `语音剪贴板同步.md` 中的设计文档。
